# 项目说明

这是一个纯前端 GitHub Pages 静态页面项目，不需要后端、数据库或构建流程。

## 文件结构

- `index.html`：版本入口页，只提供 v1/v2 跳转，不包含投注计算逻辑。
- `v1.html`：稳定旧版，保留当前固定三场计算逻辑，不要随意修改。
- `v2.html`：新版开发页，支持多场比赛、手动选择结果、信心权重和多种优化方式。
- `README.md`：项目说明，可按需更新。

## 开发规则

- 不需要 `npm install`。
- 不要引入 npm、Vue、React、Vite、后端服务或数据库。
- 页面必须能作为静态 HTML 直接打开，并能部署到 GitHub Pages。
- 修改时保持 iPhone Safari 兼容。
- v1 是稳定历史版本，除非发现明显运行错误，否则不要改动。
- v2 是主要开发页面。
- 所有投注金额必须是 2 元整数倍。
- 每个生成的串关组合至少买 1 注。
- 总投入不能超过预算上限。
- 修改后需要手动在浏览器中测试。

## 测试清单

1. 首页能打开并跳转 v1/v2；
2. v1 仍能正常使用；
3. v2 默认三场比赛；
4. v2 可以添加第四场；
5. v2 可以删除比赛；
6. 某场只选一个结果时能计算；
7. 某场选两个结果时显示信心控件；
8. 信心默认绑定赔率更低的结果；
9. 手动修改更看好结果后，不会被赔率输入反复覆盖；
10. 最低收益率模式可用；
11. 平均收益率模式可用；
12. 最高收益率模式可用；
13. 所有金额都是 2 元整数倍；
14. 每个组合至少 1 注；
15. 总投入不超过预算上限；
16. 组合数超过 256 时有提示；
17. 复制清单内容准确；
18. iPhone Safari 页面布局不崩。
